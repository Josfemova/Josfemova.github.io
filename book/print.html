Guía de uso

Protocolo

Este es un protocolo solicitud-respuesta. Se emite exactamente un mensaje de respuesta tras cada mensaje de solicitud. Cada mensaje es una secuencia de caracteres ASCII que finaliza en \r.

Solicitudes

[@aa][?|!]<cc>[p1[,p2[,p3]]]\r

  • @aa: Para comunicación con múltiples esclavos (RS-485), el prefijo opcional @aa indica la dirección decimal del destinatario, tal que los dispositivos con direcciones distintas ignoran el mensaje.

  • ?: Este prefijo opcional indica que se lea el valor actual de un parámetro, en vez de sobreescribirlo. Los parámetros que siguen al número de canal se deben omitir.

  • !: Reestablece el parámetro a su valor predeterminado. No se puede utilizar junto a ?.

  • cc: Comando. Algunos comandos esperan un número de canal, el cual se incluye en lo que sintácticamente corresponde a p1, en cuyo caso el primer argumento p1 se toma sintácticamente de p2.

  • pN: Canal y argumentos de comando.

Respuestas

Si la respuesta inicia con la secuencia #E, entonces se trata de un error. De lo contrario, es una lista de valores de respuesta.

  • #E<nnn>\r: Código de error nnn decimal.
    • #E010: Error de sintaxis. Si la solicitud es ambigua, se escoge una mejor interpretación en vez de generar un error de sintaxis.
    • #E020: Fallo general.
  • [r1[,r2[,r3]]]\r: Lista de valores de respuesta, sigue el mismo formato que los argumentos en los mensajes de solicitud.

Comandos

DT: Mostrar texto

  • Sinopsis: Imprime texto ASCII en la pantalla 4.
  • Parámetros
    • text: Cadena arbitraria de hasta 32 bytes.
  • Ejemplos de uso | Solicitud | Respuesta | Efectos | |--------------|-------------|---------------------------------| | DT,HELLO\r | \r | Se imprime "HELLO" en pantalla. | | ?DT | "HELLO"\r | | | !DT | \r | Limpia la pantalla. |
